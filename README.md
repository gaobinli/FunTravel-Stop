# 蔬菜销售电商平台

## 项目基本信息
| 维度       | 详情                |
|------------|---------------------|
| 开发角色   | 全栈开发            |
| 项目周期   | 2023.10.7 - 2024.05.1 |

## 项目描述
基于 SpringBoot+Vue3 构建的生鲜类蔬菜销售专属电商平台，采用前后端分离架构，搭建用户管理、商品管理、购物车、订单处理、评价互动、商家入驻、轮播图展示、模块分类八大核心模块，针对平台管理员、生鲜商家、普通消费者三类主体搭建差异化业务操作与权限体系。平台聚焦生鲜蔬菜销售的时效性与流通特性，打通 “用户线上下单 - 商家库存核销 - 平台运营监管” 的生鲜交易全链路，有效解决传统蔬菜销售流通环节多、库存损耗高、供需信息不对称、交易结算效率低的行业痛点。

## 核心技术栈
- 后端：Spring Boot、Spring Security、MyBatis Plus、MySQL、Druid、Swagger、Lombok、MapStruct、FastJSON、JWT、Guava、Hutool、Maven
- 前端：Vue3、Element Plus、Vue Router、Vuex

## 技术亮点
### 1. 多角色权限管理体系
采用 JWT 实现无状态认证，结合 Spring Security 为消费者、生鲜商家、平台管理员配置差异化权限矩阵；自定义请求拦截器校验接口访问合法性，构建多层级安全防护网，将系统非法访问率降至 0，权限校验接口响应延迟控制在 50ms 内。

### 2. 标准化分层架构设计
采用 Controller-Service-Mapper 经典三层架构，通过 MapStruct 组件实现 DTO、Entity、PO 数据对象的自动化转换，同时封装统一的接口返回格式与全局异常处理机制；搭建从前端页面交互到后端数据库持久化的完整数据链路，代码复用率提升 35%，系统整体可维护性增强 40%。

### 3. 生鲜电商核心功能落地
开发适配蔬菜生鲜属性的商品 SKU 管理、购物车、订单履约等功能，支持商品产地溯源展示、临期商品自动标记、订单配送状态实时跟踪、用户收货评价互动等生鲜专属能力，可稳定承接日均 800 + 笔蔬菜订单，订单履约准确率达 99.2%。

### 4. 响应式前端界面设计
基于 Element Plus 和 Vue3 搭建适配多终端的响应式界面，针对生鲜商品的展示特性优化首页轮播、分类导航、商品详情等交互模块；实现购物车智能结算、优惠券自动匹配等便捷功能，用户下单操作步骤减少 30%，整体购买转化率提升 18%。

### 5. 模块化解耦设计提升可维护性
按生鲜电商业务域对用户、商品、订单等模块进行解耦拆分，通过 RESTful API 规范前后端数据通信；该架构支持商家入驻功能的快速迭代，新功能上线周期缩短 50%，同时为后续接入生鲜配送物流模块预留了灵活的扩展接口。
